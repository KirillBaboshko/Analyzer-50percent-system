# Руководство по внесению вклада

Спасибо за интерес к проекту Dota 2 Analyzer! Мы рады любому вкладу.

## 🚀 Как начать

1. **Форкните репозиторий**
2. **Клонируйте свой форк**
   ```bash
   git clone https://github.com/your-username/dota2-analyzer.git
   cd dota2-analyzer
   ```
3. **Создайте ветку для фичи**
   ```bash
   git checkout -b feature/amazing-feature
   ```

## 💻 Разработка

### Backend (Go)

```bash
cd dota-analyzer
go run cmd/main.go
```

### Frontend (React)

```bash
cd dota-analyzer-frontend
npm install
npm run dev
```

## 📝 Стандарты кода

### Go
- Следуйте [Effective Go](https://golang.org/doc/effective_go)
- Используйте `go fmt` для форматирования
- Запускайте `go vet` перед коммитом
- Добавляйте комментарии к экспортируемым функциям

### TypeScript/React
- Используйте TypeScript strict mode
- Следуйте [React Best Practices](https://react.dev/learn)
- Используйте функциональные компоненты и хуки
- Именуйте компоненты в PascalCase
- Именуйте файлы так же, как компоненты

### Стиль коммитов

Используйте [Conventional Commits](https://www.conventionalcommits.org/):

```
feat: добавить экспорт результатов в CSV
fix: исправить расчет винрейта для анонимов
docs: обновить README с примерами
style: отформатировать код
refactor: переписать компонент MatchCard
test: добавить тесты для useAnalysis
chore: обновить зависимости
```

## 📤 Отправка Pull Request

1. Убедитесь, что код компилируется без ошибок
2. Добавьте тесты для новой функциональности
3. Обновите документацию, если необходимо
4. Создайте Pull Request с описанием изменений

### Шаблон PR

```markdown
## Описание
Краткое описание изменений

## Тип изменений
- [ ] Исправление бага
- [ ] Новая функциональность
- [ ] Улучшение производительности
- [ ] Рефакторинг
- [ ] Документация

## Чеклист
- [ ] Код компилируется без ошибок
- [ ] Добавлены/обновлены тесты
- [ ] Обновлена документация
- [ ] Следует стандартам кода проекта
```
